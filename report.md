## Задание 1: Консольные приложения в REPL.it

### Вариант 5
**Условие:** Дан массив натуральных чисел размерности n. Вывести на печать все числа этого массива, являющиеся полными квадратами. Функция должна проверять, является число полным квадратом или нет.

### Код программы (`src/main.c`)

```c
/*
 * Автор: Вороненко Константин
 * Группа: 12
 * Описание: Программа находит полные квадраты в массиве натуральных чисел.
 */

#include <stdio.h>
#include <stdlib.h>
#include <math.h>

int isPerfectSquare(int number) {
    if (number < 0) return 0;
    int root = (int)sqrt(number);
    return (root * root == number) ? 1 : 0;
}

void printPerfectSquares(int arr[], int size) {
    printf("Полные квадраты в массиве:\n");
    int found = 0;
    for (int i = 0; i < size; i++) {
        if (isPerfectSquare(arr[i])) {
            printf("%d (квадрат числа %d)\n", arr[i], (int)sqrt(arr[i]));
            found = 1;
        }
    }
    if (!found) printf("Полные квадраты не найдены.\n");
}

int main(int argc, char *argv[]) {
    if (argc < 2) {
        printf("Использование: %s <размер> <числа...>\n", argv[0]);
        return 1;
    }

    int size = atoi(argv[1]);
    if (size <= 0) {
        printf("Ошибка: Размер массива должен быть положительным\n");
        return 1;
    }

    int *numbers = (int*)malloc(size * sizeof(int));
    printf("Исходный массив:\n");
    for (int i = 0; i < size; i++) {
        numbers[i] = atoi(argv[i + 2]);
        printf("%d ", numbers[i]);
    }
    printf("\n\n");

    printPerfectSquares(numbers, size);
    free(numbers);
    return 0;
}
```

```bash

### 3. Что такое «сборка»?
**Ответ:** Сборка — процесс преобразования исходного кода в исполняемый файл, включающий компиляцию, линковку и другие этапы.

### 4. Какие утилиты могут использоваться для сборки?
**Ответ:** make, cmake, gcc, g++, automake, ninja.

### 5. Для чего служит Makefile?
**Ответ:** Makefile — файл с инструкциями для утилиты make, описывающий правила сборки проекта, зависимости между файлами и команды компиляции.

### 6. Что такое цель в Makefile? Приведите пример.
**Ответ:** Цель — это имя, которое можно указать при вызове make для выполнения определённых действий.
```makefile
run: $(TARGET)
	$(TARGET)
```

### 7. Что такое связка в Makefile? Приведите пример.
**Ответ:** Связка (rule) — это цель + зависимости + команды.
```makefile
$(TARGET): $(OBJECTS)
	gcc -o $(TARGET) $(OBJECTS)
```

### 8. Что такое зависимость в Makefile? Приведите пример.
**Ответ:** Зависимости — файлы, которые должны существовать до выполнения цели.
```makefile
main.o: main.c
	gcc -c main.c
```
Здесь `main.c` — зависимость.

### 9. Что такое правило в Makefile? Приведите пример.
**Ответ:** Правило описывает, как создать цель из зависимостей.
```makefile
%.o: %.c
	gcc -c $< -o $@
```

### 10. Что такое макроопределение в Makefile? Приведите пример.
**Ответ:** Макроопределение — переменная в Makefile.
```makefile
CC = gcc
CFLAGS = -Wall
```

### 11. Какая связка используется для очистки проекта?
**Ответ:** `clean` — удаляет все сгенерированные файлы.
```makefile
clean:
	rm -rf $(OBJ_DIR) $(BIN_DIR)
```

### 12. В чем заключается модель КИС для проектов на языке С?
**Ответ:** Модель КИС (Клиент-Интерфейс-Сервер):
- **Клиент** — main.c (использует функции)
- **Интерфейс** — .h файлы (объявления функций, структур)
- **Сервер** — .c файлы (реализация функций)

### 13. Что такое стандарты кодирования и почему рекомендуется им следовать?
**Ответ:** Стандарты кодирования — правила оформления кода (отступы, комментарии, именование). Они обеспечивают читаемость, поддерживаемость и единообразие кода в команде.

### 14. Какой должна быть структура проекта согласно модели КИС и правилам сборки?
**Ответ:**
```
project/
├── bin/           # исполняемые файлы
├── obj/           # объектные файлы
├── src/           # исходный код (.c)
├── include/       # заголовочные файлы (.h)
├── docs/          # документация
└── Makefile       # сборка
```

### 15. Что такое Github Actions и для каких задач применяется данный функционал?
**Ответ:** GitHub Actions — инструмент CI/CD для автоматизации сборки, тестирования и развёртывания проектов при пушах и pull request'ах.

### 16. Что такое workflow в GitHub Actions? Приведите пример.
**Ответ:** Workflow — автоматизированный процесс, описанный в YAML-файле.
```yaml
name: CI
on: [push]
jobs:
  build:
    runs-on: ubuntu-latest
```

### 17. Что такое event в GitHub Actions? Приведите пример.
**Ответ:** Event — событие, запускающее workflow (push, pull_request, schedule).
```yaml
on:
  push:
    branches: [ main ]
```

### 18. Что такое job в GitHub Actions? Приведите пример.
**Ответ:** Job — набор шагов, выполняемых на одном runner'е.
```yaml
jobs:
  build:
    runs-on: ubuntu-latest
```

### 19. Что такое step в GitHub Actions? Приведите пример.
**Ответ:** Step — отдельная задача внутри job.
```yaml
steps:
  - name: Checkout
    uses: actions/checkout@v4
```

### 20. Что такое action в GitHub Actions? Приведите пример.
**Ответ:** Action — переиспользуемый модуль для выполнения задачи.
```yaml
- uses: actions/checkout@v4
```
